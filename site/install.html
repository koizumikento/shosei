shosei

Japanese publishing workflows for EPUB, PDF, Kindle, and print handoff.

Overview Install Usage Config Quickstart Contributing GitHub

Install

tag 付き GitHub Release では CLI archive と VSIX を配布し、Homebrew / Scoop 向け manifest は package repository publish が通ったときに更新します。source install も引き続き使えます。

Requirements

  • macOS では Homebrew、Windows では Scoop、または source install 用の Rust toolchain と `cargo`
  • 対象 platform: macOS / Windows / Linux

Install with Homebrew (macOS)

brew tap koizumikento/stray-tools https://github.com/koizumikento/stray-tools.git
brew install koizumikento/stray-tools/shosei

Install with Scoop (Windows)

scoop bucket add stray-tools https://github.com/koizumikento/stray-tools
scoop install stray-tools/shosei

Install from source

リポジトリを clone して `shosei` バイナリを install します。

git clone https://github.com/koizumikento/shosei.git
cd shosei
cargo install --path crates/shosei-cli

Release archives

GitHub Release には `x86_64-unknown-linux-gnu`, `x86_64-apple-darwin`, `aarch64-apple-darwin`, `x86_64-pc-windows-msvc` の CLI archive と VSIX が載ります。Linux では `x86_64-unknown-linux-gnu` archive をそのまま展開して使えます。

Smoke check

インストール前に CLI surface を確認するだけなら、次で十分です。

cargo run -p shosei-cli --bin shosei -- --help

What gets installed

GitHub Release archive は release asset の `shosei` binary を使います。Homebrew / Scoop は release 後に package repository publish が成功した場合、その archive を参照する formula / manifest から導入できます。

`cargo install --path crates/shosei-cli` は workspace 内の CLI crate から `shosei` バイナリを build して install します。
